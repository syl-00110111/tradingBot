===============================================================================
                 WORKFLOW, CONCEPTS ET PARAMÈTRES FIXES DU BOT
===============================================================================

Ce document détaille le fonctionnement interne du bot, les concepts de validation
utilisés et documente de manière exhaustive les paramètres numériques fixes
("hardcodés") identifiés dans le code source.

1. CONCEPTS CLÉS ET LEURS RELATIONS
-----------------------------------
A. BACKTEST (Test Rétrospectif)
   - DÉFINITION : Simulation de trading sur données historiques.
   - RELATION AVEC LE BENCHMARK : Le backtest est l'unité de mesure élémentaire.
     Le benchmark utilise des milliers de backtests pour comparer les stratégies.
   - RELATION AVEC MONTE CARLO : Le backtest donne un profit brut. Monte Carlo
     teste si ce profit est reproductible ou s'il n'est dû qu'à la chance.

B. BENCHMARK (Étalonnage / Auto-Optimisation)
   - DÉFINITION : Processus de recherche globale au démarrage du bot.
   - FONCTIONNEMENT : Le bot teste systématiquement les 36 stratégies sur chaque
     paire configurée pour trouver la plus performante sur le terme choisi.

C. VALIDATION MONTE CARLO (Robustesse Statistique)
   - DÉFINITION : Simulation de 100 à 1000 "univers parallèles" où le prix
     bouge de manière aléatoire mais réaliste.
   - RÔLE : Si une stratégie échoue dans 50% des univers parallèles, elle est
     jugée risquée, même si son backtest était excellent.

2. WORKFLOW ACTUEL DU BOT (Flux de travail)
-------------------------------------------
1. PHASE DE DÉCOUVERTE : Le bot charge les marchés et détecte le matériel
   (GPU/CPU).
2. PHASE D'AUTO-OPTIMISATION (Benchmark) :
   - Scan massif de l'historique (10 000 bougies).
   - Calcul des 4 meilleurs "Success Patterns" (modèles de réussite).
   - Calcul du B.Prof final (Profit * Score Monte Carlo).
3. PHASE DE SURVEILLANCE :
   - Boucle infinie d'analyse des paires (analyze_pair).
   - Si la forme du prix actuel ressemble à un "Success Pattern" (> 70%) :
     -> La stratégie du pattern est activée.
4. PHASE D'EXÉCUTION :
   - Si un signal d'achat est confirmé : execute_buy.
   - Si un signal de vente est confirmé : execute_sell.
   - Les achats sont priorisés selon le B.Prof (le plus haut d'abord).

3. DOCUMENTATION EXHAUSTIVE DES CHIFFRES "HARDCODÉS"
----------------------------------------------------

A. BOT.PY (Cœur du système)
   - 10 000 : Nombre max de bougies historiques récupérées par paire (L.1719).
   - 4 : Nombre maximum de "Success Patterns" conservés. Ce sont les fenêtres
     temporelles les plus rentables et non-chevauchantes (L.1615).
   - 500 : Taille du tampon de bougies pour la stabilité des indicateurs (L.813).
   - 100 : Nombre de simulations Monte Carlo pendant le benchmark (L.1636).
   - 20 : Nombre de bougies d'horizon pour la simulation Monte Carlo (L.1636).
   - 0.70 : Seuil de similarité pour déclencher une stratégie (L.831).
   - 0.015 : Profit minimum pour enregistrer un pattern pendant le scan (L.1576).
   - 0.022 : Seuil de profit pour le message "No successful patterns" (L.1835).
   - 100 EUR : Capital virtuel de départ pour les backtests internes (L.1390).
   - 10 EUR : Montant minimum de sécurité pour un ordre (protection Notional).
   - 0.001 (0.1%) : Taux de frais par défaut si l'API ne répond pas (L.1316).
   - 1.0 (100%) : Multiplicateur Monte Carlo neutre (L.1447).
   - 0.8 / 0.5 / 0.2 : Coefficients de pondération de récence (L.1584-1589).

B. INDICATORS.PY (Stratégies et Détection)
   - 1000 : Nombre de simulations Monte Carlo en analyse temps réel (L.98).
   - 0.7 / 0.3 : Pondération Similarité (70% Prix / 30% Technique) (L.198).
   - 8 / 18 : Périodes EMA "Fast/Slow" par défaut (L.117-118).
   - 14 : Période RSI et ADX par défaut (L.123-124).
   - 3 : Coefficient de détection des baleines (3x l'écart-type) (L.144).
   - 0.001 : Seuil de détection du "Range" (écart EMA/Prix < 0.1%) (L.153).

C. MONTE_CARLO.PY (Moteur Statistique)
   - 5000 : Nombre de simulations par défaut de la classe (L.19).
   - 100 : Nombre de bougies d'horizon par défaut de la classe (L.19).
   - 0.0015 (0.15%) : Frais attendus lors de la validation de profit (L.64).
   - 0.5 : Base du score Monte Carlo (Score = 0.5 + Probabilité de profit) (L.67).

D. TRADING_ENGINE.PY (Gestion des risques)
   - 1.2 : Multiplicateur de risque global par défaut (L.9).
   - 17.0 : Montant de transaction de base par défaut (L.35).
   - 2 : Seuil de série de victoires (win streak) pour bonus (L.41).
   - 1.3 : Multiplicateur de bonus en cas de série de victoires (L.42).
   - 25 : Seuil ADX pour passer en mode "Tendance Forte" (L.15).
   - 0.015 : Seuil de volatilité pour passer en mode "Haute Volatilité" (L.22).
